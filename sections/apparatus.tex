    \smallskip
      Every 12 h, a node can:
      \begin{enumerate}
        \item \textbf{Failsafe} --- runway \textcolor{tud Orange}{< 2 days}
          \(\to\) sweep remaining funds to the richest peer.
        \item \textbf{Top-up} --- runway < \textcolor{tud Orange}{30 days} \(\to\) buy VPS credits to cover another 30 days
        \item \textbf{Spawn} --- if post-spawn runway enough \(\to\) provision and fund a child.
        \item \textbf{Idle} --- wait for the next tick.
      \end{enumerate}

      \medskip
      \textcolor{tud Orange}{Identical code runs on every node.}
    \end{column}

{
\setbeamerfont{itemize/enumerate body}{size=\small} % template pins lists to \normalsize
\begin{frame}{Reproduction mechanics}
  \small
  \begin{block}{Check post-spawn total runway}
    A node's \textbf{total runway} \(=\) VPS days \(+\) SporeStack credit
    \(+\) BTC. A node spawns only if it is left with \(\geq 60\times(1+c)\) days of runway after spawning, where \(c\) is the caution value.
  \end{block}

  \begin{itemize}
    \item A child starts with a \textbf{30-day} pre-paid VPS and inherits
      \textbf{40\%} of the parent's remaining Bitcoin.
    \item Actual Pipeline: mutate caution \(\to\) provision VPS \(\to\) fresh BTC wallet \(\to\) deploy code over SSH \(\to\) transfer inheritance.
  \end{itemize}

  \medskip
  \medskip
  \medskip
  \textcolor{tud Orange}{Nodes must remain with significant runway \textbf{after} spawning a child.}
\end{frame}
}
